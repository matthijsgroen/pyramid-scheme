Feature Delta — Rich World Redesign

Pyramid Scheme

One map primitive, three nested scales — and at every scale you reveal a hidden picture by doing math.

The redesign replaces a flat sequence of puzzle tableaux with a navigable world: every pyramid becomes a place you return to, every tomb a source of power that reshapes the map behind you. What follows describes what changed and why it matters.

Was — the flat loop

An expedition is a linear queue of pyramid levels. Each level is one tableau puzzle followed by another. Loot drops are probabilistic; tomb access is a threshold check. Visiting a site once uses it up.

Now — the rich world

Each pyramid is a navigable interior — corridors, fog, branches. Every site holds dormant content waiting for a key you haven't found yet. The world grows backward as you push forward.

World Structure

One primitive, three nested scales

The same underlying model — nodes connected by corridors, fog burning back on entry, an explorer dot as wayfinding — runs at every zoom level. Build it once.

I Journey map Pyramid and tomb sites as nodes along a linear path — fog reveals the next site on completion
II Pyramid interior A node graph of rooms and corridors — puzzle, fork, gate, treasure, stairhead, exit, warning
III Tableau puzzle The math encounter itself — unchanged engine, six families now, time-of-day chooses the instrument
Was Three distinct, unrelated screens — expedition list, level picker, tableau — with no shared visual language between them.

Pyramid Interiors

Fog, corridors, and a map that is its own progress bar

The interior is an SVG/DOM node graph laid on a grid. Fog burns back as you navigate. The map is the only progress indicator needed.

  • Seven node types — all configs of one primitive: puzzle, fork (free branch reveal), seal gate, ward gate, treasure, stairhead, exit. A warning node precedes every trap corridor so entering is always a choice.
  • Three cell states — fogged / revealed-but-locked (visible through a gate, not yet reachable) / reachable. The glint of a treasure behind a locked gate is an invitation, not an accident.
  • Explorer dot as camera — it follows your taps along the corridor route (~250ms), never blocking input. It anchors the viewport on large floors and makes exploration feel directional: the corridor extends from where you stand.
  • Stairheads descend, not cut — the dot drops to the floor below and the camera follows. A floor change is a move.
  • Multi-floor pyramids — deeper floors unlock gradually; each floor is a self-contained math-band zone so a dormant lower floor can be revisited independently.
Was Each level was a full-screen tableau with a progress bar and a "next" button. No interior space, no navigation.

Non-Linear Time

Nothing is spent — sites stay dormant until the right key arrives

Two gate types create two rhythms: one that resolves in a single visit, one that forces a return trip weeks later.

Gate type Key source What it gates Effect on the world
Seal A chest inside this same site Optional branch within one visit Fully solvable on first contact; no revisit needed
Ward A tomb treasure from another tier A floor or branch locked until that tomb is cleared Makes early sites dormant — alive again when the key arrives later
  • Ward etiquette — every ward announces itself at the fork (a silhouette: "tomb-warded corridor that way") so committing to a branch is always an informed choice.
  • Critical path always open — wards may gate optional branches; the main path through any site is completable with keys available at that point in the journey.
  • "What just opened" badges — when a tomb key lands, a per-journey badge lists which dormant rooms across all sites are newly reachable. The data is a free byproduct of the validator's forward pass.
  • Journey hub after first completion — complete a journey once and it becomes a fast-travel hub: every site a direct tap-target. The completion flag is the switch; first-run pacing is never compromised.
Was No revisits. Completing a level marked it done. Expeditions were one-way; there was no reason to return to an earlier pyramid.

Reward Economies

Three economies at three scopes — all deterministic

Loot is authored, not rolled. Each economy operates at a distinct scale and never collapses into a single priority chain.

Hieroglyph fragment
Room
308 fragments across the game; 2–8 per hieroglyph (scaled by tier and tomb section). Completing a hieroglyph unlocks a tomb tableau room. Permanent once found. No two fragments of the same glyph in the same journey.
Map piece
Journey
One authored placement per tomb per journey — not a random drop. Surface floors feed the first tomb; deep floors (opened by earlier ward keys) feed later tombs. Collecting enough pieces opens the tomb gate.
Mosaic tile
Game
298 tiles, one stained-glass picture. The critical path through any pyramid always awards tiles; optional branches add more. A half-built mosaic is a to-do list. The completed picture is the natural ending.

Math System

Six puzzle families — time of day chooses the instrument

All families share the same DNA: hidden values recovered by deduction. What changes is which Egyptian instrument dresses the puzzle — and that choice is mechanical, not cosmetic.

Cross-sum Addition and substitution. The pyramid tableau. Present from tier 1, deepens throughout.
Balance scale Equality and solve-for-x. The weighing of the heart against Ma'at's feather.
Sundial Reading analog time. A value source — its answer flows downstream as a known symbol. Day expeditions only.
Water-clock Elapsed time, duration, boundary-crossing subtraction. The clepsydra draining between marks.
Eye of Horus fractions Unit fractions (½ to 1/64). The Wedjat eye parts as the fraction system.
Clock arithmetic Modular arithmetic mod 12/24. Ritual durations wrapping the day. Higher tiers and decoy rooms.
  • Carry-forward is mechanical — a sundial reading made upstream may be consumed downstream as a known symbol. Carry-forward never crosses a stairhead, so each floor is a self-contained dependency zone.
  • Time of day picks the family — morning/afternoon/evening/night selects which instruments appear. A night expedition swaps the sundial for the decan star-clock variant.
  • Family is orthogonal to position — which math runs inside a puzzle node and where that node sits on the map are separate dimensions. A hard floor can contain a simple water-clock; an easy branch can hold fractions.
Was Cross-sum only, with time-of-day as a cosmetic theme (background art) rather than a mechanical choice. No carry-forward between rooms.

Survival Layer — Expert tier and above

Traps, health, and the decision to press on

Trap corridors introduce a second kind of encounter: opt-in, time-pressured, and consequential without being punishing.

  • Warning node — a free node placed before every trapped corridor. The player sees the trap ahead and may turn back. Entering is always a deliberate choice.
  • Timed math encounter — moving through a trapped corridor triggers a timed question in the current puzzle family. Pass to proceed; fail to take health damage. The clock matches the family (sundial questions have a longer window; fractions a shorter one).
  • Health is session-persistent — 3 base hearts, expandable to 6 with tomb upgrades. Depleted health blocks further trap attempts until restored.
  • Consumables in normal chests — mummy bandages (restore ½ heart), healing oils (restore 1 heart), trap tools (bypass a trap without answering). ~147 consumables authored across the game.
  • Trap corridor as optional branch — the reward behind the trap is always an optional branch endpoint (mosaic tiles, deep map piece). The critical path is never trapped.
Was No health system. Every puzzle was equivalent — no optional risk/reward decision, no consequence for errors beyond blocking progression on that tableau.

Tombs & Permanent Power

Nine tombs, forty treasures, a player who grows across the whole game

Every tomb treasure does two things: it opens a specific dormant pyramid floor somewhere in the world, and it permanently changes how the player experiences future content.

Tomb count by tier

  • Starter — 1 tomb, 4 treasures
  • Junior — 1 tomb, 6 treasures
  • Expert — 2 tombs, 4 + 4 treasures
  • Master — 2 tombs, 5 + 5 treasures
  • Wizard — 3 tombs, 4 + 4 + 4 treasures

Treasure types

  • 36 ward keys — each opens one specific pyramid floor
  • 4 location keys — reveal the next tomb in the same tier
  • 17 of 40 carry a passive perk; the remaining 23 are pure keys (the floor they unlock is the reward)
Max health +½ heart
Raises the health ceiling; 6 half-heart upgrades total. Caps at 6 hearts.
Armor / protection
Reduces trap damage by 1 half-heart per stack. 2 upgrades; caps at 1 full heart reduction.
Trap insight +1s
Adds 1 second to every trap time limit. 2 upgrades; caps at +2s total.
Hieroglyph compass L1–3
L1: which pyramid holds a tracked fragment · L2: proximity signal inside the site · L3: exact path to the chest.
Hidden path detection L1–4
L1: hidden corridors pulse during exploration · L2: floor indicator · L3: pyramid marker · L4: journey list marker.

Early tombs grant health upgrades (immediate, universal power). Higher-tier tombs grant compass and detection (situational utility that reshapes navigation). Hard caps across all categories prevent snowballing.

Was 5 tombs (one per tier). Treasure effects were passive modifiers to the probabilistic loot system — two of the five effect types (higherLootChance, mapFragmentChance) become vacuous once the redesign's deterministic rewards land.

Generation & Authoring

Authored constraints, seeded decoration

The generator assembles from authored budgets; the seed only decides cosmetic variation within those bounds. Difficulty is never an accident of which seed ran.

  • Budgets-plus-pins authoring — specify puzzle count, branch factor, gate types, and reward placement as a budget. Pin specific nodes that must sit in a specific position. Stone-tier pyramids need zero pins; complex late sites need three or four.
  • Solvable by construction — the assembler generates a reachability order first, then places each lock on an edge and drops its key anywhere in the already-reachable subtree. A generated layout cannot be unsolvable.
  • Machine-discarded bad seeds — a forward-pass validator runs on every candidate. Seeds that fail (gate before key, mosaic stranded behind a ward, all-bland fork) are discarded automatically, never by eye. Failure is always explained, never a silent re-roll.
  • Staggered ramps — structural complexity and math difficulty climb independently. Every new structural verb (first fork, first gate, first second floor) debuts in easy-math territory. Early pyramids are the tutorial, diegetically.
